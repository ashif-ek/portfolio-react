# Security Policy

This document describes how security issues are handled for the **Portfolio React** project.

---

## Supported Versions

This project is a **personal portfolio application** and does not provide long-term or versioned security support.

Only the **latest code available on the `main` branch** is considered current.

| Version | Supported |
|--------|-----------|
| Latest (main) | ✔️ |
| Older commits | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

- **Do not** open a public GitHub issue
- Contact the maintainer directly via:
  - GitHub profile: https://github.com/ashif-ek

### What to Include

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Screenshots or logs (if helpful)

---

## Response Expectations

- Initial response: **within 3–7 days**
- Valid vulnerabilities will be reviewed and addressed as appropriate
- This project does not offer bug bounties or formal SLAs

---

## Scope

This policy applies only to:
- Source code in this repository
- Publicly deployed portfolio instances

It does **not** cover:
- Third-party services
- Hosting providers
- Browser or platform vulnerabilities

---

## Acknowledgement

Responsible disclosure is appreciated and helps improve the quality of this project.
